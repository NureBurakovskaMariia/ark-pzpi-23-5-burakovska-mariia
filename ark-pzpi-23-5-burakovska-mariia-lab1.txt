     3 ОПИС ВИКОНАНОЇ РОБОТИ

     У ході виконання лабораторної роботи було розроблено документ у форматі Vision & Scope (Додаток Б) для програмної системи «Система допомоги тваринам у притулках з використанням веб- та мобільного застосунку». Метою роботи було дослідити проблему нестачі цифрових інструментів для управління притулками, описати концепцію майбутнього рішення, визначити його бізнес-логіку, ключові функції та технологічну базу для реалізації.
     Було виконано такі кроки:
* проаналізовано основні проблеми, з якими стикаються притулки: ручний облік тварин, відсутність прозорої комунікації з волонтерами та донорами, складність управління пожертвами;
* проведено аналіз аналогічних рішень (Adopets, Adopt a Pet, тощо) та визначено їхні обмеження — відсутність української локалізації, неможливість детального ведення медичної історії тварини, відсутність інтеграції волонтерських процесів;
* сформовано опис призначення системи — створення єдиного цифрового простору для управління тваринами, волонтерами, усиновленням та пожертвами;
* визначено бізнес-логіку: реєстрація користувачів, додавання тварин, оформлення запиту на усиновлення, перегляд потреб притулків, надсилання донатів, базова панель адміністратора;
* описано архітектуру системи: серверна частина, веб-клієнт, мобільний клієнт та емуляція IoT (у майбутньому — можливість підключення датчиків для моніторингу стану тварин);
* обрано стек технологій: React (JavaScript) для веб-клієнта, Kotlin (Android, Android Studio) для мобільного застосунку, Node.js + Express (JavaScript) для серверної частини, SQLite як проста файлова база даних для початкової реалізації, GitHub для контролю версій, Postman для тестування API, Python-скрипти для емуляції роботи IoT-клієнта.
     Таким чином, було створено повний документ Vision & Scope, який відповідає вимогам завдання та може бути використаний як основа для подальшої розробки системи в наступних лабораторних роботах.

     1.1. Аналіз проблеми
   У сучасних притулках для тварин часто виникають труднощі з ефективним управлінням ресурсами, контролем умов утримання, обліком тварин і координацією волонтерів.
   Багато процесів виконуються вручну або ведуться у таблицях, що призводить до:
* втрати або дублювання інформації про тварин;
* відсутності системного моніторингу стану притулку;
* труднощів у комунікації між адміністрацією та волонтерами;
* низької прозорості у звітності щодо донатів.
   Це знижує ефективність роботи та негативно впливає на якість догляду за тваринами.
Тому виникає потреба у комплексній програмній системі, яка дозволить автоматизувати основні процеси, забезпечити аналітику та контроль умов утримання.
     1.2. Аналіз аналогів
   	Існуючі рішення (наприклад, ShelterBuddy, PetPoint, RescueGroups) надають окремі функції — облік тварин або управління волонтерами — але не підтримують моніторинг середовища та інтеграцію з IoT.
   	Основні недоліки аналогів:
* відсутність інтеграції між обліком тварин і умовами утримання;
* складні інтерфейси або висока вартість;
* обмежений доступ через мобільні пристрої.
   	Запропонована система покращить ситуацію за рахунок:
* єдиної бази даних для адміністраторів, волонтерів і користувачів;
* веб- та мобільного клієнтів з інтуїтивним інтерфейсом;
* модулю-емулятора IoT, який генерує тестові дані про температуру й вологість у приміщенні притулку, імітуючи роботу сенсорів.
2.1. Опис призначення системи
	Система призначена для:
* обліку тварин (дані, медична історія, вакцинації, статус усиновлення);
* управління волонтерами, завданнями та графіками;
* ведення обліку донатів та матеріальної допомоги;
* моніторингу умов утримання тварин через модуль-емулятор IoT;
* взаємодії з користувачами, які бажають усиновити тварину.
	Цінність для користувачів:
* для адміністрації — автоматизація обліку й аналітики;
* для волонтерів — зручне планування роботи;
* для громадян — швидкий доступ до інформації про тварин і можливість допомоги притулку.
     2.2 Опис бізнес-логіки
Основні бізнес-процеси системи:
* адміністратор або працівник притулку додає інформацію про нову тварину;
* система зберігає дані у центральній базі;
* IoT-емулятор передає тестові показники середовища до сервера;
* сервер обробляє дані, формує записи у базі та, при потребі, генерує сповіщення (наприклад, про підвищену температуру в клітці);
* через веб-або мобільний застосунок користувачі зможуть переглядати інформацію, редагувати її, отримувати повідомлення та статистику.
     3.1 Опис функціональності серверної частини та IoT-клієнта
     Серверна частина:
* прийом і обробка запитів від IoT-емулятора;
* зберігання даних у базі (інформація про тварин, показники датчиків, журнали подій);
* авторизація користувачів (адміністраторів, волонтерів);
* надання REST-API для клієнтських застосунків.
     IoT-клієнт (програмна симуляція):
* генерує тестові дані про стан умов у клітках;
* відправляє ці дані на сервер через HTTP-запити або MQTT-протокол;
* отримує відповіді про статус або сповіщення від сервера.
     3.2 Опис функціональності веб-клієнта та мобільного клієнта
     Веб-клієнт:
1. перегляд списку тварин і стану кліток;
2. додавання або редагування записів;
3. візуалізація даних сенсорів (графіки, таблиці);
4. управління волонтерами та завданнями;
5. облік і керування донатами:
1) сторінка прийому донатів (онлайн-формат для платежів / інструкції для переказів);
2) реєстрація матеріальних пожертв (їжа, препарати, інвентар);
3) генерація звітів по донатах;
4) прив’язка донатів до конкретних потреб/кампаній (наприклад, збір на лікування тварини).
     Мобільний клієнт:
1. швидкий доступ до бази тварин;
2. отримання push-сповіщень (наприклад, про потребу у волонтерській допомозі чи важливі оновлення);
3. можливість пошуку тварини для прилаштування;
4. перегляд останніх оновлень і статусів;
5. функції донатів для користувачів:
1) швидка форма для одноразових пожертв та інструкції для матеріальної допомоги;
2) перегляд історії власних пожертв та отриманих подяк/квитанцій.

     4.3 Опис програмних засобів та інструментів
     Розробка програмної системи для підтримки діяльності притулків тварин передбачає використання технологій, які забезпечують простоту розробки, легкість розгортання, можливість подальшого розширення та інтеграцію з мобільним застосунком і модулем IoT-емуляції.
     Клієнтська частина веб-системи створюватиметься з використанням бібліотеки React мовою JavaScript. Такий вибір забезпечує мінімальний поріг входу для розробки, модульність, швидке оновлення інтерфейсу та можливість подальшого масштабування. Інтерфейс буде адаптивним, що дозволить коректно відображати веб-додаток як на комп’ютерах, так і на мобільних пристроях.
     Серверна частина реалізовуватиметься на платформі Node.js з використанням фреймворку Express. Такий підхід дозволяє швидко створювати REST API, обробляти користувацькі запити, працювати з базою даних, виконувати автентифікацію та авторизацію. Використання JavaScript на бекенді спрощує розробку, оскільки одна мова використовується і на клієнті, і на сервері.
     Для зберігання даних обрано SQLite — легку файлову реляційну базу даних, яка не потребує окремого серверного середовища. Вона є зручною для навчальних проєктів, легко переноситься, підтримує SQL-запити та забезпечує збереження інформації про тварин, волонтерів, донати, запити користувачів та інші дані системи.
     Мобільний клієнт розроблятиметься мовою Kotlin у середовищі Android Studio. Це забезпечує нативну продуктивність, підтримку архітектури MVVM та бібліотек Android Jetpack. Мобільний застосунок дозволятиме переглядати інформацію про тварин, залишати заявки на допомогу, здійснювати донати, отримувати сповіщення від притулку.
     Для імітації роботи IoT-пристроїв застосовуватимуться Python-скрипти, які генерують тестові дані (наприклад, зміну температури у вольєрі або наявність руху) та надсилають їх на сервер через HTTP-запити. Такий спосіб дозволяє демонструвати функціональність майбутньої системи без використання реальних сенсорів або мікроконтролерів.
     Контроль версій коду здійснюватиметься за допомогою Git із використанням платформи GitHub для збереження репозиторію. Для тестування API застосовується Postman. Розробка інтерфейсу та серверної частини виконуватиметься у Visual Studio Code, тоді як для мобільного клієнта використовуватиметься Android Studio.
